## 用户角色与权限

系统采用基于角色的访问控制（RBAC），支持三种用户角色：

| 角色 | 权限范围 | 适用场景 |
|------|---------|---------|
| **普通用户** | 查看自己的账号、任务、兑换记录 | 日常运营人员 |
| **管理员** | 管理所有用户、任务、兑换配置、公告 | 团队负责人 |
| **超级管理员** | 系统配置、审计日志、所有管理员权限 | IT 管理员 |

### 权限矩阵

| 功能模块 | 普通用户 | 管理员 | 超级管理员 |
|---------|---------|--------|-----------|
| 仪表盘 | 查看 | 查看 | 查看 |
| 账号管理 | 查看/编辑自己的账号 | 查看/编辑所有账号 | 查看/编辑所有账号 |
| 任务管理 | 查看/管理自己的任务 | 查看/管理所有任务 | 查看/管理所有任务 |
| 兑换中心 | 查看/创建兑换 | 查看/管理所有兑换 | 查看/管理所有兑换 |
| 数据导出 | 导出自己的数据 | 导出所有数据 | 导出所有数据 |
| Webhook 管理 | 查看自己的 Webhook | 查看/管理所有 Webhook | 查看/管理所有 Webhook |
| 系统设置 | 修改个人信息 | 查看系统设置 | 完整系统配置 |
| 审计日志 | 无权限 | 查看操作日志 | 查看/导出审计日志 |

## 快速上手

### 首次登录

1. 访问系统首页：`http://localhost`
2. 使用默认管理员账号登录：
   - 用户名：`admin`
   - 密码：首次登录后需立即修改
3. 登录后进入仪表盘，查看系统概览

### 修改密码

1. 点击右上角头像 → 个人设置
2. 选择"修改密码"
3. 输入当前密码和新密码
4. 点击"确认修改"

> **安全提示**：建议使用至少 8 位字符，包含大小写字母、数字和特殊字符的强密码。

## 功能模块详解

### 📊 仪表盘

系统首页，提供关键指标概览。

**功能说明**：

| 指标 | 说明 |
|------|------|
| 系统健康状态 | API、Worker、数据库、Redis 连接状态 |
| 账号总数 | 已添加的云盘账号数量 |
| 今日签到 | 今日自动签到成功/失败数量 |
| 任务队列 | 待执行、执行中、已完成任务数量 |
| 兑换成功率 | 近 7 日兑换成功率趋势 |
| 公告通知 | 最新系统公告和通知 |

**操作**：
- 点击指标卡片查看详情
- 点击图表区域钻取数据
- 刷新按钮获取最新数据

### 👤 账号管理

管理多个云盘服务账号。

**功能说明**：

| 操作 | 说明 |
|------|------|
| 添加账号 | 录入云盘平台账号信息（用户名、密码、平台） |
| 编辑账号 | 修改账号信息、状态、备注 |
| 删除账号 | 删除不再使用的账号 |
| 批量导入 | 支持 CSV 批量导入账号 |
| 状态监控 | 查看账号在线状态、最后签到时间 |

**操作流程**：

1. 点击左侧菜单"账号管理"
2. 点击"添加账号"按钮
3. 填写账号信息：
   - 平台：选择云盘平台类型
   - 用户名：云盘账号用户名
   - 密码：云盘账号密码
   - 备注：可选备注信息
4. 点击"保存"

**批量导入示例 CSV**：

```csv

caiyun,user002,pass002,Account group A
caiyun,user003,pass003,Account group B
```

### ⏰ 任务中心

管理定时任务和自动化操作。

**功能说明**：

| 操作 | 说明 |
|------|------|
| 创建任务 | 设置任务名称、类型、执行时间、关联账号 |
| 查看任务 | 查看任务列表、状态、执行历史 |
| 启停控制 | 启用/禁用任务执行 |
| 删除任务 | 删除不再需要的任务 |

**任务类型**：

| 类型 | 说明 | Cron 示例 |
|------|------|-----------|
| 签到任务 | 自动签到云盘账号 | `0 8 * * *`（每天 8 点） |
| 兑换任务 | 自动兑换指定产品 | `0 10 * * *`（每天 10 点） |
| 检查任务 | 检查账号状态 | `0 */6 * * *`（每 6 小时） |

**操作流程**：

1. 点击左侧菜单"任务中心"
2. 点击"创建任务"按钮
3. 填写任务信息：
   - 任务名称：自定义任务名称
   - 任务类型：选择签到/兑换/检查
   - 执行时间：Cron 表达式
   - 关联账号：选择要关联的账号
4. 点击"保存"

配置事件通知和外部系统集成。

**功能说明**：

| 操作 | 说明 |
|------|------|
| 创建端点 | 创建 Webhook 接收端点 |

**支持的事件类型**：

| 事件类型 | 触发时机 |
|---------|---------|
| `exchange.completed` | 兑换任务完成 |
| `exchange.failed` | 兑换任务失败 |
| `exchange.created` | 兑换任务创建 |
| `task.completed` | 任务执行完成 |
| `task.failed` | 任务执行失败 |
| `account.created` | 账号创建 |
| `account.updated` | 账号更新 |

1. 点击左侧菜单"Webhook 管理"
2. 点击"创建端点"按钮
3. 填写端点信息：

   - URL：接收 Webhook 的 URL（HTTPS 推荐）
   - 事件：选择要订阅的事件类型
4. 复制生成的 Secret（仅显示一次）
5. 点击"保存"
6. 在目标系统中配置签名验证

## 操作流程示例

### 示例 1：批量导入账号并创建签到任务

1. **准备账号数据**

   创建 `accounts.csv` 文件：
   ```csv
   platform,username,password,remark
   caiyun,user001,pass001,Group A
   caiyun,user002,pass002,Group A
   caiyun,user003,pass003,Group B
   ```

2. **批量导入账号**
   - 进入"账号管理"
   - 点击"批量导入"
   - 上传 CSV 文件
   - 确认导入

3. **创建签到任务**
   - 进入"任务中心"
   - 点击"创建任务"
   - 任务名称：每日签到
   - 任务类型：签到任务
   - Cron 表达式：`0 8 * * *`（每天 8 点）
   - 关联账号：选择 Group A 的所有账号
   - 点击"保存"

4. **启用任务**
   - 在任务列表中找到刚创建的任务
   - 点击"启用"按钮

---

### 示例 2：配置 Webhook 通知

1. **创建 Webhook 端点**
   - 进入"Webhook 管理"
   - 点击"创建端点"
   - 名称：兑换完成通知
   - URL：`https://your-server.com/webhook`
   - 事件：选择 `exchange.completed`、`exchange.failed`
   - 点击"保存"
   - **复制 Secret**（仅显示一次）

2. **配置接收端**

   ```python
   # Flask 接收示例
   from flask import Flask, request
   import hmac
   import hashlib

   app = Flask(__name__)
   SECRET = "whsec_xxxxxxxxxxxxxxxxxxxx"

   @app.route('/webhook', methods=['POST'])
   def webhook():
       signature = request.headers.get('X-Caiyun-Signature')
       timestamp = request.headers.get('X-Caiyun-Timestamp')
       body = request.get_data(as_text=True)

       # 验证签名
       expected = hmac.new(
           SECRET.encode(),
           f"{timestamp}\n{body}".encode(),
           hashlib.sha256
       ).hexdigest()

       if not hmac.compare_digest(expected, signature):
           return 'Invalid signature', 403

       # 处理事件
       event = request.json
       print(f"Received event: {event}")
       return 'OK', 200
   ```

3. **测试投递**
   - 在 Webhook 列表中点击"测试"
   - 查看投递日志确认成功

## 常见问题排查

### 1. 账号添加失败

**可能原因**：
- 账号密码错误
- 网络连接超时
- 云盘服务端限流

**解决方法**：
- 检查账号密码是否正确
- 稍后重试
- 查看任务日志获取详细错误信息

---

### 2. 兑换任务未执行

**可能原因**：
- 任务未启用
- 到达执行时间但队列繁忙
- 账号 Cookie 过期

**解决方法**：
- 检查任务状态是否为"启用中"
- 查看队列监控页面
- 更新账号 Cookie

---

### 3. 导出文件下载失败

**可能原因**：
- 导出任务未完成
- 文件已过期（7 天自动清理）
- 磁盘空间不足

**解决方法**：
- 等待任务完成后再下载
- 重新创建导出任务
- 联系管理员清理磁盘

---

### 4. Webhook 接收失败

**可能原因**：
- 目标 URL 无法访问
- HMAC 签名验证失败
- 响应状态码非 2xx

**解决方法**：
- 检查目标服务是否可访问
- 验证 Webhook Secret 配置
- 查看投递日志中的详细错误信息

---

### 5. 任务执行超时

**可能原因**：
- 网络延迟过高
- 目标服务响应慢
- 任务并发过多

**解决方法**：
- 调整任务执行时间避开高峰
- 减少并发任务数量
- 联系管理员检查系统资源

---

### 6. 登录失败

**可能原因**：
- 用户名或密码错误
- 账户被锁定（多次登录失败）
- 系统维护中

**解决方法**：
- 确认用户名和密码正确
- 等待账户解锁（通常 15 分钟）
- 联系管理员确认系统状态

## 最佳实践

### 账号管理

- 为不同用途的账号添加清晰的备注
- 定期检查账号状态，及时处理异常账号
- 不要复用密码，使用密码管理器生成强密码

### 任务管理

- 合理设置任务执行时间，避免高峰期
- 为重要任务启用失败重试
- 定期查看任务执行日志，及时发现异常

### 数据安全

- 定期修改登录密码
- 不要在公共电脑上保存登录状态
- 及时导出重要数据并安全存储

### Webhook 使用

- 使用 HTTPS 端点
- 妥善保管 Webhook Secret
- 实施幂等性处理，防止重复投递

## 术语表

| 术语 | 说明 |
|------|------|
| 账号 | 云盘服务登录凭证，包含用户名、密码、平台信息 |
| 任务 | 定时执行的自动化操作，如签到、兑换 |
| 兑换 | 使用兑换码兑换云盘服务产品 |
| Webhook | 事件通知机制，系统主动向外部 URL 推送事件 |
| Cron | 定时任务表达式，定义任务执行时间 |
| Token | 身份认证令牌，用于 API 请求认证 |
| JWT | JSON Web Token，一种开放标准令牌格式 |
